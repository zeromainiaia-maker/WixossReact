# semantic audit スケールアップ 第2弾（残り clean群80枚）トリアージ（PLAN.md §3 Sonnetタスク8・2026-07-16・続き144・Sonnet 5）

対象＝続き102（2026-07-12）が `claude -p` セッション上限で中断した stub100+clean100（seed 202607・perGroup 100・batchSize 10）サンプルのうち、
未監査だった残り80枚（すべて clean群＝全効果 AUTO かつ STUB 無し＝逆翻訳の対照群）。

## 再開時の注意（重要・後続セッション向け）
アーカイブ `scripts/archive/scratchpad/semantic_audit_101/` には `findings.jsonl`/`findings_compact.txt`/`manifest.json` のみが残り、
バッチ完了判定に使う `prompts/`/`raw/` は元の作業ディレクトリ（gitignore対象）ごと消失していた（続き139で指摘済みの問題）。
さらに **`--per-group 100 --seed 202607` を素朴に再実行しても元の200枚と一致しない**ことを実測確認した
（母集団のstub/clean境界がその後の census-batch 等で185/200件ドリフト＝再現不能）。
今回は**アーカイブ済み `manifest.json` の `picked` 配列（順序固定・200枚）をそのまま真実のソースとして `picked.slice(120,200)` で残り80枚を明示指定**
（`semanticAuditExtract.mjs --cards <80枚CSV>`）して再現性を確保した。次回このサンプルを追監査する場合も同様に `manifest.json` の `picked` を直接参照すること（再サンプリングしない）。

## 結果
80枚中70枚を一括実行→1バッチ（batch_01・10枚）がAPIサーバーエラーで失敗→単独再実行で回収。全80枚完走。
**findings 88件（HIGH 57 / MED 28 / LOW 3）・指摘ありカード 43/80枚**。
アーカイブの `findings.jsonl` に旧125件（続き102・batch1-12・119枚分）と統合し**累計213件**（`batch`フィールドは元の通し番号13-20へ補正済み）。
これで seed=202607 の200枚サンプル（stub100+clean100）は**全数監査完了**。

## サンプル精度メモ
HIGH findingsから無作為に12件超をJSON本体と原文で直接照合＝**全件で指摘内容が正確**（owner/count/duration/timing/条件節の欠落を具体的フィールド名まで正しく特定）。
既知の偽陽性パターン（STUB+CONDITIONAL(IS_MY_TURN)任意コストイディオム／LIFE_BURST mandatory:false／アンコール・ベット注記）はclean群のためほぼ該当なし。
低精度が疑われるのはLOW 3件のみ（うち1件はLLM自身が「意味的には合致している可能性が高い」と留保付き＝WXDi-P01-030）。

### B. duration/until の誤り＝一時効果が永続化 or 即時化
