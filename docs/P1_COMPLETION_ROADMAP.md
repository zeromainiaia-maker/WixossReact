## このドキュメントの位置づけ
- **P1完了判定のための残作業マップ**。`npm run census` の高シグナル欠落効果（現在 BASELINE_HIGH=1817・明細 `docs/_vocab_census.txt`）を機械分類し、「parser表現で直せる純P1」「機構待ち（§6.3送り）」「両方が混在」の3区分に仕分けた結果を記録する。
- **2026-07-22 時点のスナップショット（効果単位）**。数字は census 再生成（`npm run census`）のたびに更新される。再現手順は末尾「再現方法」参照。
- P1完了は「census高シグナルを0にする」ことではない。PLAN §5「完了判定」の定義＝**高シグナルが逓減限界に達したら「P1完了＋残りは機構待ち」と宣言し、機構待ちを §6/§7（P2/P3）へ送る**。本ドキュメントはその宣言に向けて「残りのうち何が本当に機構待ちか／何がまだ parser 作業か」を可視化する。

## 重要な注意（caveat）
1. **粗い網はまだ残る（PLAN §3 原則「件数メトリクスを信じない」）**：効果単位化で旧最大の死角（同カード別効果のマスキング）は解消したが、「小さい数373」「数値不一致146」は数値の別表現（CHOOSE内・countIsX等）を拾えない粗い網のまま＝偽陽性を含む。ただし**この2カテゴリだけに掛かる効果は 114件**で、仮に全部偽陽性でも全体の 5% にすぎない。
2. **効率の勘所**：系統カテゴリ（条件節611・クラス389・色133・閾値184）は「1 parser規則 → 多数効果」で減らせる。手法は続き106/107/143で実証済み＝`STATE_CONDITION_CLAUSES` への条件追加・`parseHoistStateCondition`・`parseThisWayGenericCount`・`parseBareBranchCondition` の横展開。
3. **census外の残作業もある**：本マップは census 計器の可視範囲のみ。census が原理的に見えない残作業（参照解決の誤り・engine実装差・実行順序）は PARTIAL刻印計器（152件）・semantic audit findings（タスク12(xxvii)(xxix)）・BEHAVIOR_AUDIT が補完する（PLAN §5c「残死角」）。

## 再現方法（数字の更新手順）
- **入力**：`docs/_vocab_census.txt`（`npm run census` で再生成）。
- **手順**：
  1. 各 `## カテゴリ ［原文該当 X／高シグナル N／…］` の `### 高シグナル（対応語彙なし）` 直後の行が効果IDリスト。
  2. トークンを正規化：`(...)` サフィックス除去（`.replace(/\(.*\)$/,'')`）＋「（なし）」プレースホルダ除外→重複除外（この粒度で vocabCensus の headline と一致する）。
  3. 上記「機構キーワード集合」で各カテゴリを P1／機構に二分。
  4. 効果ごとに「機構カテゴリのみ→純§6.3／P1カテゴリのみ→純P1／両方→混在」で集計。
  5. カード単位ロールアップが要る場合のみ `-E\d+`/`-BURST`/`-LAYER` 等の効果サフィックスを除去。
- ワンショットスクリプトは scratchpad の使い捨て（`p1class_effect.mjs`・2026-07-16）＝上記手順から再構成可能。
