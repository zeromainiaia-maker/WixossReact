## 重要な注意（caveat）
1. **粗い網はまだ残る（PLAN §3 原則「件数メトリクスを信じない」）**：効果単位化で旧最大の死角（同カード別効果のマスキング）は解消したが、「小さい数285」「数値不一致125」は数値の別表現（CHOOSE内・countIsX等）を拾えない粗い網のまま＝偽陽性を含む。ただし**この2カテゴリだけに掛かる効果は 115件**で、仮に全部偽陽性でも全体の 6% にすぎない。
2. **効率の勘所**：系統カテゴリ（条件節488・クラス272・色110＋共通色73・閾値143）は「1 parser規則 → 多数効果」で減らせる。手法は続き106/107/143で実証済み＝`STATE_CONDITION_CLAUSES` への条件追加・`parseHoistStateCondition`・`parseThisWayGenericCount`・`parseBareBranchCondition` の横展開。
3. **census外の残作業もある**：本マップは census 計器の可視範囲のみ。census が原理的に見えない残作業（参照解決の誤り・engine実装差・実行順序）は PARTIAL刻印計器（152件）・semantic audit findings（タスク12(xxvii)(xxix)）・BEHAVIOR_AUDIT が補完する（PLAN §5c「残死角」）。

### 純P1から外へ出したもの（再編時の除外判断）
- **「代わりに」75＝カテゴリごとトリアージ側へ**（分類表の備考「一部はタスク6」より強い扱い）。バニッシュ先変更・コスト代替・リコレクト・イベント/コスト参照など単発§6.3が主体。`LAST_PROCESSED_MATCHES`＋加算モデルで表せると確認できた効果だけ個別に純P1へ戻す（WX25-P2-102/107/109 の前例＝続き235b）。
- **引用能力付与62**＝permanent・相手付与・STUB能力が混在するため一括再収穫しない。duration/triggerScope を既存 action に足すだけで済む効果を個別確認して戻す。
- **カテゴリ横断の相対値・履歴条件**＝census の機構カテゴリに掛からなくても engine state が要るものは除外（例：WX10-029-E2「このシグニのパワーの半分以下」。**動的比較14件のキーワードに漏れあり**＝「半分以下」「自身のパワー以下」等を分類キーワードへ追加する余地）。
- **「機構カテゴリに掛からない＝純P1」の機械判定をそのまま実装判断に使わない**こと（上例のような取りこぼしが実在）。

### 粗い網2カテゴリのサンプリング所見（各10件・等間隔抽出＝統計的推定ではない）
- **数値不一致125**：真陽性 約9/10 と高精度。偽陽性例＝WX25-P2-102-E1（原文-8000 を JSON が -5000＋チャーム時追加-3000 に分解済み＝意味保持）。「数値が無い」だけで断定せず加算分解・`countIsX`・条件内値を確認する。
- **小さい数285**：真陽性 約6/10・偽陽性 約3/10（15/25 等の部分一致を誤検知）・機構混在 1/10 ⇒ **総数285を作業量として使わない**。既定方針「粗い網のみ115効果の単発精査」が妥当。

## 再現方法（数字の更新手順）
- **入力**：`docs/_vocab_census.txt`（`npm run census` で再生成）。
- **手順**：
