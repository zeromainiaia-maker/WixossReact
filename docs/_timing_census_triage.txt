# timing センサス残クラスタの振り分け台帳（2026-07-17・Sonnet・PLAN §3 補欠(a)）
#
# 目的＝docs/_timing_census.txt の各クラスタについて、engine に該当timingの収集関数（collector）が
# 既に存在するかを機械照合し、「Opusタスク16（parser regex 1本で直る＝parser語彙のみ不足）」と
# 「§6.3送り（収集関数/filter軸が無く新規機構が要る）」に仕分ける。
#
# 判定基準（3段階）:
#   [A] 完全wired　＝該当ON_*の収集関数が存在し、必要なtriggerScope/triggerFilter/triggerConditionの
#       軸も型に既にある。parser側でtiming/scope/filterを抽出するregexを足すだけで直る＝Opusタスク16。
#   [B] 部分wired　＝収集関数（イベント検出）自体はあるが、必要なfilter/閾値/scope軸が型に無い。
#       parser regexだけでは直らず、TargetFilter/triggerCondition等への軽量engine拡張が要る。
#   [C] 未wired　＝該当イベントを検出する収集関数が存在しない（0参照）か、scope/方向が根本的に逆
#       （例：相手起因イベントを自分側で監視する経路が無い）。新規機構＝§6.3送り。
#
# 根拠は grep（`triggerCollect.ts`/`BattleScreen.tsx`/`effects.ts`のON_*コメント）による実地確認。
# 総数: 117クラスタ（`npm run census:timing`実行時点）。⚠この母数はセンサス再実行のたびに微増減する
# （継続的なIS_MY_TURN化消化等の副作用）ので、実数より「各クラスタの判定ロジック」を再利用すること。
#
# ---------------------------------------------------------------------------
